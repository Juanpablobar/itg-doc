# Cómo Publicar una Página en Nuestro Sitio de Documentación (Devs)

Este documento proporciona una guía paso a paso para publicar una página en nuestro sitio de documentación utilizando Nextra y GitHub.

## Requisitos Previos

Asegúrate de tener lo siguiente antes de comenzar:

1. Una cuenta de GitHub y pertenecer a la organización de ITGlobers.
2. Acceso al [repositorio de GitHub](https://github.com/ITGlobers/ITG-Docs) donde se aloja nuestro sitio de documentación.

## Pasos para Publicar una Página

### Paso 1: Crear o Editar un Archivo `.mdx`

1. Clona el repositorio de GitHub a tu entorno local.
2. Navega a la carpeta correspondiente donde deseas añadir o editar una página.
3. Crea un nuevo archivo `.mdx` o edita uno existente.

### Paso 2: Redactar el Contenido

1. Utiliza Markdown para redactar el contenido de tu página.
2. Puedes incluir componentes JSX si es necesario.

### Paso 3: Previsualizar la Página

Antes de subir tu página, es importante previsualizarla para asegurarte de que todo se muestra correctamente. Puedes usar herramientas en línea como:

- [StackEdit](https://stackedit.io/): Un editor en línea que permite ver cómo se renderiza el Markdown.
- [Dillinger](https://dillinger.io/): Otro editor Markdown en línea con vista previa.

Estas herramientas son útiles para previsualizar contenido Markdown, pero ten en cuenta que pueden no soportar todas las características específicas de `.mdx`.

### Paso 4: Subir los Cambios a GitHub

1. Añade y haz commit de tus cambios en tu entorno local.
2. Haz push de los cambios al repositorio de GitHub.

### Paso 5: Verificación del Despliegue

Una vez que hagas push de tus cambios, el sistema de CI/CD se encargará del despliegue automático de tu página. Despues de unos minutos verifica en el sitio de documentación que tu página se haya desplegado correctamente.

## Ayuda Adicional
